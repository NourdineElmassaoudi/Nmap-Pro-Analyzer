🚀 QUICK START (5 MINUTES):
══════════════════════════════════════════════════════════════════════════════

STEP 1: Install Dependencies
─────────────────────────────
$ pip install python-nmap rich

If you get permission errors:
$ pip install python-nmap rich --break-system-packages


STEP 2: Test with Demo
──────────────────────
$ python nmap_pro_analyzer.py --demo

This shows a sample network analysis with:
  ✓ Windows Domain Controller
  ✓ Linux Web Servers
  ✓ Mail Server
  ✓ Docker Host
  ✓ NAS/Storage
  ✓ Network Device


STEP 3: Analyze Your Own Network
─────────────────────────────────
$ sudo nmap -sV -O -p- -oX output.xml <your-target>
$ python nmap_pro_analyzer.py output.xml


That's it! Now you'll see:
  ✓ All hosts (up/down)
  ✓ Operating system for each host
  ✓ All open ports and services
  ✓ Service versions detected
  ✓ Risk levels (Critical, High, Medium, Low)
  ✓ Inferred server roles (Web Server, Database, AD, etc.)
  ✓ Summary statistics

⚙️ COMMON NMAP COMMANDS:
══════════════════════════════════════════════════════════════════════════════

Quick service scan (10 minutes):
  $ sudo nmap -sV -p- -oX quick.xml 192.168.1.100
  $ python nmap_pro_analyzer.py quick.xml

Full enumeration with OS detection (20 minutes):
  $ sudo nmap -sV -O -p- -oX full.xml 192.168.1.100
  $ python nmap_pro_analyzer.py full.xml

Network sweep (your subnet):
  $ sudo nmap -sV -O -oX network.xml 192.168.1.0/24
  $ python nmap_pro_analyzer.py network.xml

Critical ports only (fast):
  $ nmap -sV -p 22,80,443,445,389,88,3306,5432 -oX critical.xml <target>
  $ python nmap_pro_analyzer.py critical.xml

For more options → See QUICK_REFERENCE.md


✨ WHAT THE ANALYZER WILL TELL YOU:
══════════════════════════════════════════════════════════════════════════════

For each host you scan, you'll get:

1. Host Header
   ├─ IP Address
   ├─ Hostname(s)
   ├─ Status (UP/DOWN)
   ├─ Operating System (if detected)
   └─ Number of open ports

2. Port Table
   ├─ Port Number
   ├─ Protocol (TCP/UDP)
   ├─ State (open/closed/filtered)
   ├─ Service Name
   ├─ Product & Version
   └─ Risk Level (color-coded)

3. Role Inference
   └─ What this host appears to be:
      ├─ Web Server
      ├─ Database Server
      ├─ Windows Domain Controller
      ├─ Mail Server
      ├─ Docker Host
      └─ (And 45+ other possibilities)

4. Summary Table
   └─ Overview of all scanned hosts:
      ├─ IP addresses
      ├─ Hostnames
      ├─ Operating systems
      ├─ Open port counts
      ├─ Critical port counts ⚠️
      └─ Inferred roles

⚠️ RED FLAGS TO WATCH FOR:
══════════════════════════════════════════════════════════════════════════════

CRITICAL (Fix immediately):
  ⚠️ Database ports open to internet (3306, 5432, 1433)
  ⚠️ Domain Controller with only partial AD ports open
  ⚠️ RDP accessible from internet (3389)
  ⚠️ Telnet or unencrypted protocols (port 23)

HIGH (Investigate soon):
  ⚠️ SSH with old versions (e.g., OpenSSH 5.x)
  ⚠️ Multiple critical ports on the same host
  ⚠️ Unnecessary services running
  ⚠️ No firewall protection on critical services

MEDIUM (Plan to address):
  ⚠️ Outdated web server versions
  ⚠️ No HTTPS enforcement
  ⚠️ Unpatched services


🆘 TROUBLESHOOTING:
══════════════════════════════════════════════════════════════════════════════

Q: Command not found error
A: Make sure you're in the right directory:
   $ cd /path/to/nmap_pro_analyzer/
   $ python nmap_pro_analyzer.py scan.xml

Q: "python-nmap is not installed"
A: Install it:
   $ pip install python-nmap rich

Q: "File not found"
A: Make sure the XML file exists:
   $ ls -la scan.xml
   Or provide full path:
   $ python nmap_pro_analyzer.py /full/path/to/scan.xml

Q: No ports shown for a host I know has open ports
A: Your scan might not have checked those ports.
   Rescan with full enumeration:
   $ nmap -sV -O -p- -oX scan.xml <target>

Q: How do I check if a service is vulnerable?
A: Take the version and search online:
   "Service Name Version CVE"
   Example: "MySQL 5.5.40 CVE"
